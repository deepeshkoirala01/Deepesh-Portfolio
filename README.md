---------------------------------------------------------------------------------------------------------------------------
Project Summary
---------------------------------------------------------------------------------------------------------------------------
This repository contains my Chada Tech Clock Application, which was developed as module three Project One assignment for 
CS 210: Programming Languages. The application displays both a 12‑hour clock and a 24‑hour clock simultaneously while 
keeping both clocks synchronized. Users can enter an initial time and update it by adding one hour, one minute, or one 
second through a menu‑driven interface. The program uses modular functions, input validation, and proper rollover logic
to ensure accuracy and reliability.

---------------------------------------------------------------------------------------------------------------------------
Portfolio Reflection Questions
---------------------------------------------------------------------------------------------------------------------------

Question 1: Summarize the project and what problem it was solving.

The purpose of this project was to create a C++ application that displays both a 12‑hour and a 24‑hour clock side‑by‑side.
The program solves the problem of maintaining accurate, synchronized time in two different formats while allowing users to
update the time through a menu‑driven interface. The project emphasizes modular programming, input validation, and proper
rollover logic to ensure the clocks remain accurate and reliable. This meets the needs of companies like Chada Tech that
serve international clients following ISO 8601 time formatting standards.


Question 2: What did you do particularly well?

I believe one of the strongest aspects of this project was organizing the program into separate functions, each with a
specific responsibility. This made the code easier to read, debug, and maintain. I also implemented accurate rollover
logic so that hours, minutes, and seconds updated correctly and both clock formats always remained synchronized.
Additionally, I used meaningful variable names, consistent formatting, and comments throughout the code to improve
readability.


Question 3: Where could you enhance your code? How would these improvements make your code more efficient, secure, and
so on?

Although the project meets all of the required functional requirements, there are several improvements I could make. I
would redesign the application using a dedicated Clock class to better demonstrate object‑oriented programming principles.
I would also improve input validation by detecting non‑numeric input instead of only checking numeric ranges. Separating
the program into multiple source and header files would improve maintainability, and adding features such as displaying 
the current system time or automatically updating the clocks would enhance the user experience. These improvements would
make the application more scalable, maintainable, and reliable.


Question 4: Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or
resources are you adding to your support network?

The most challenging part of the project was implementing the rollover logic for seconds, minutes, and hours. Updating one
unit of time could affect another, so I needed to ensure that every function worked together correctly. I overcame this
challenge by breaking the problem into smaller functions, testing each function individually, and verifying boundary cases
such as midnight. Throughout the project, I relied on the course materials, ZyBooks, Visual Studio's debugging tools, and
C++ documentation. These resources have become valuable additions to my support network and have also helped me with future 
programming projects.


Question 5: What skills from this project will be particularly transferable to other projects or course work?

This project strengthened several skills that I will continue to use throughout my Computer Science program. These include
modular programming, writing reusable functions, validating user input, debugging, testing edge cases, and organizing code
in a logical manner. I also improved my problem‑solving abilities by breaking a larger programming task into smaller,
manageable components. These skills will be valuable in future software development courses and professional programming
projects.


Question 6: How did you make this program maintainable, readable, and adaptable?

I made the program maintainable by dividing related tasks into separate functions with clearly defined responsibilities.
I used descriptive variable and function names, consistent formatting, and comments to improve readability. The modular
structure also makes the program adaptable because additional features or menu options can be added with minimal changes
to the existing code. This design makes the application easier to understand, maintain, and extend over time.
